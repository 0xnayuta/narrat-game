# 外部参考策略（DoL / 其他仓库）

## 目标
在不污染当前架构边界的前提下，引入外部项目作为**专题参考源**，而不是默认实现答案。

## 参考目录（当前指定）

- DoL 外部源码参考目录：
  - DoL 外部源码参考目录：`../degrees-of-lewdity`
- 用途：仅用于阅读、检索和机制提炼。
- 禁止：将该目录代码直接复制到本项目运行路径。

## 强制规则

1. **只读参考**
   - 外部参考仓库（如 DoL）仅用于阅读与分析。
   - 当前 DoL 参考目录为 `../degrees-of-lewdity`。
   - 不将外部仓库源码直接纳入本项目运行路径。

2. **禁止直接复制代码**
   - 禁止将参考仓库代码片段直接粘贴到本项目。
   - 允许提炼机制、数据结构思路与流程，再按本项目边界独立实现。

3. **先提炼，后实现**
   - 每次参考实现前，先写出最小“机制提炼说明”：
     - 参考了什么问题域（例如事件优先级/任务推进）
     - 提炼出的规则是什么
     - 如何映射到当前项目边界（engine/content/ui/tests）

4. **小步改动 + 可回归验证**
   - 一次只落一个小目标。
   - 每次改动必须配套最小测试或回归命令，保证可回退。

5. **边界不破坏原则**
   - 保持 `content` / `engine` / `ui` 分层。
   - UI 不承载核心规则逻辑。
   - 不因参考而引入与当前阶段无关的大抽象或大依赖。

3. 存档演进策略（迁移注册与 schema 膨胀控制）

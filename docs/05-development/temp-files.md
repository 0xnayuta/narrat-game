# 临时产物策略

## 规则

1. **临时测试文件** → `tests/tmp/`，不在仓库提交
2. **临时输出文件** → `tmp/`，不在仓库提交
3. **文档草稿** → 写完后直接提交或移到 `docs/_archive/`，不留未归档草稿
4. **TODO / FIXME** → 保留在代码中作为已知问题标记，不要删除（它们是问题清单的来源）

## 临时目录说明

- `tests/tmp/` — 临时测试文件（.test.cjs）
- `tmp/` — 临时输出、构建产物

这两个目录通过 `.gitignore` 排除，不提交到仓库。

## 不在此列的临时产物

| 产物 | 处理方式 |
|---|---|
| Debug print / console.log | 随功能代码一起清理或移除 |
| 废弃的测试用例 | 删除，不要留在注释里 |
| 过期的文档草稿 | 移到 `docs/_archive/` 或删除 |
| 未提交的代码改动 | 评估后提交或回退，不要留在 worktree 太久 |
